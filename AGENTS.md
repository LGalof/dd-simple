# AGENTS

## Purpose

This repository is a professional university software project scaffold for D&D Simple.

## Working Agreements

- Keep the repository organized by domain and package boundaries.
- Prefer small, reviewable changes with updated documentation when architecture changes.
- Record significant technical decisions in `docs/adr/`.
- Keep business features out of the initial scaffold unless explicitly requested.
- Keep the backend API, frontend UI, and shared package decoupled and well documented.

## Coding Expectations

- Use TypeScript across frontend, backend, and shared packages.
- Keep code readable, modular, and easy for a student team to maintain.
- Add tests alongside features in later iterations.
