# ADR-004 Websocket Realtime

## Status

Proposed

## Context

Realtime collaboration is expected to become important for session support and gameplay synchronization.

## Decision

Reserve a future architectural path for WebSocket-based realtime communication, but do not implement it in the initial scaffold.

## Consequences

- initial setup stays simpler
- backend remains ready for later expansion
- realtime design will be addressed in a later iteration
