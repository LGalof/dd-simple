# 01 Project Plan

## Delivery Approach

The project will be delivered incrementally through iterations suitable for university coursework.

## Early Milestones

1. Initial repository and architecture setup
2. Character and session domain modeling
3. Backend persistence and API growth
4. Frontend workflows and interaction flows
5. Testing, deployment, and final presentation

## Team Practices

- maintain documentation in `docs/`
- review major changes before merging
- keep architecture decisions recorded as ADRs
